### 1.10 Warehouse (Backend) vs Location (Frontend)

O frontend mantem tipos `Location`/`ApiLocation`/`CreateLocationRequest` marcados como `@deprecated`, sugerindo migracao para Warehouse/Zone/Bin. O backend tem schemas separados para `warehouse`, `zone` e `bin`, mas o frontend `warehouse.types.ts` nao tem interfaces `Warehouse`, `Zone` ou `Bin`.

| Status   | Detalhe                                                                                                                    |
| -------- | -------------------------------------------------------------------------------------------------------------------------- |
| **WARN** | Frontend `warehouse.types.ts` contem apenas tipos legados (Location). Faltam interfaces Warehouse, Zone e Bin no frontend. |

> Os hooks `use-stock-other.ts` usam `useLocations()` apontando para `/v1/locations`. Nao ha hooks para warehouses, zones ou bins no frontend stock.

## 5. Shape de Paginacao (WARN)

| Aspecto             | Backend (common.schema)      | Frontend (PaginationMeta)              | Status                      |

> O frontend tem `normalizePagination()` que aceita tanto `pages` quanto `totalPages` e `hasPrevious` nao esta mapeado (espera `hasPrev`). Deveria aceitar ambos.

## 7. Campos Opcionais (WARN)

### 7.1 `nullable()` vs `optional` (undefined)

O backend usa `z.xxx().nullable()` para campos que podem ser `null` no banco de dados. O frontend frequentemente usa `field?: Type` (optional), que aceita `undefined` mas nao `null`.
